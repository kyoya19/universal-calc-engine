# v1.0 Reporting Surface Consolidation Entry

## Purpose

This document defines the first v1.0 reporting surface consolidation boundary.

The v1.0 entry does not add UI framework behavior, CSS behavior, Android native behavior, or runtime target substitution. It only records the current public reporting surface and fixes the intended public entry points with a thin export test.

## Public reporting surface

The current public reporting surface is exported from `packages/core/src/index.ts` through `packages/core/src/report_model.ts`.

The stable public types are:

- `ReportModel`
- `ReportSection`
- `ReportRow`
- `ReportRowStatus`

The stable public formatter is:

- `formatReportModelPlainText()`

The current stable public `ReportModel` adapters are:

- `generatedTargetComparisonReportToReportModel()`
- `generatedTargetSolverGateResultSummaryToReportModel()`

## Source data boundary

The generated-target comparison adapter consumes the generated-target comparison report data produced by the generated-target solver adapter layer.

The generated-target solver gate summary adapter consumes the generated-target solver gate summary data produced by the generated-target solver adapter layer.

UI consumers should read structured `ReportModel` sections, rows, statuses, classes, and metadata directly. They should not reparse existing plain text formatter output to recover row identity, status, or metadata.

## Runtime boundary

This v1.0 entry preserves the v0.9 runtime boundary:

- solver target remains `transition.to`,
- contribution target remains `transition.to`,
- `generatedTo` remains validation / diagnostics / summary / comparison report data only,
- `generatedTo` is not used as a solver target,
- `generatedTo` is not used as a contribution target,
- runtime target substitution remains unimplemented,
- `generated_candidate_as_solver_target` remains rejected as runtime policy,
- expected reward baseline remains unchanged.

## Out of scope

The following are outside this v1.0 entry:

- UI framework implementation,
- CSS implementation,
- Android native implementation,
- TeX rendering changes,
- runtime target substitution,
- connecting `generatedTo` to solver target,
- connecting `generatedTo` to contribution target,
- changing existing formatter output contracts.

## Verification

This entry is verified by a thin public surface test that imports the reporting types, formatter, and current adapters from `../src` rather than from internal file paths.

The test fixes the current barrel export boundary while leaving runtime behavior to the existing report model and generated-target solver gate tests.
