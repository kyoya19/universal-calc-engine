## Validation semantics fixed

The v4.5 tests fix the first validation layer for the production-input fixture stub:

- Requirement statuses must be one of:
  - `required`
  - `stub`
  - `deferred`
  - `excluded`
- State identifiers must be unique.
- The fixture stub must have exactly one start state.
- The fixture stub must have exactly one terminal state.
- Transition identifiers must be unique.
- Transition `fromStateId` and `toStateId` must reference existing states.
- Transition `probabilityRef` must reference an existing probability row.
- Transition `rewardRef` must reference an existing reward row.
- Probability rows must remain placeholder/stub rows.
- Reward rows must remain placeholder/stub rows.
- `generatedTo` remains deferred/null.
- Runtime target substitution remains excluded.
- Solver target remains `transition.to`.
- Contribution target remains `transition.to`.
- Report identity remains generic generated-target comparison.
