# v9.0 Juo unit metadata entry

## Status

Entry checkpoint for the Juo unit metadata phase.

Current baseline:

- #166 v8.3 citation metadata completion merged.
- Merge commit: `5100b9f9877dd70cb24f739c392965291ae86708`.

## Purpose

This phase defines the minimum unit metadata boundary that must exist before any source-backed Juo candidate can move toward a production value row.

The phase covers probability unit metadata and reward unit metadata only.

It does not accept any real Beast King / Juo probability or reward value.

## Unit metadata scope

A probability unit metadata row should record:

- unit metadata id
- source id
- source type
- value domain
- probability unit kind
- probability expression status
- denominator status
- observation basis status
- normalization status
- review status
- execution eligibility

A reward unit metadata row should record:

- unit metadata id
- source id
- source type
- value domain
- reward unit kind
- reward expression status
- payout basis status
- currency or token basis status
- normalization status
- review status
- execution eligibility

## Required separation

Unit metadata is not a production value.

A candidate with unit metadata may still be:

- non-executable
- unverified
- conflicted
- missing a citation-backed extracted value
- missing a source-backed probability numerator or denominator
- missing a source-backed reward amount
- waiting for normalization rules
- waiting for production graph binding

## Boundary

This checkpoint is documentation-only.

No real probability values, reward values, production fixture values, expected value assertions, executable production graph, runtime target substitution, report adapter changes, formatter changes, contribution output changes, UI changes, TeX changes, CSS changes, Android native changes, or Sugoroku baseline changes are included.
