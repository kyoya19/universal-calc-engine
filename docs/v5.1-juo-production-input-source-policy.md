## Purpose

This document fixes the source, citation, unit, inference, formula, and conflict policy that must be satisfied before any real Beast King / Juo production probability or reward value can be promoted from placeholder or stub status into an executable production-input row.

The policy is intentionally value-free. It defines how values may be accepted later, not what the values are.

## Row status policy

Every later production-input row must carry an explicit status before it can participate in execution.

Allowed statuses:

- `required`: the row or field is required but not yet supplied.
- `stub`: the row is a deliberate placeholder and must remain non-executable.
- `deferred`: the row is intentionally postponed and must remain non-executable.
- `excluded`: the row is intentionally outside the current model scope.
- `candidate`: the row has candidate data but is not yet accepted for execution.
- `source_backed`: the row is backed by accepted source metadata but may still be non-executable if a conflict or dependency is unresolved.
- `formula_backed`: the row is represented by an expression and dependency list.
- `inferred`: the row is derived from one or more source-backed, formula-backed, or candidate rows.
- `accepted`: the row is permitted to be executable.

Only `accepted` rows may be used as executable production probability or reward values.

## Source type policy

Every later real or candidate value must classify its source type.

Allowed source types:

- `primary`: official manual, official specification, manufacturer-published material, patent/public specification when directly relevant, or other direct official source.
- `secondary`: compiled machine-value source, guide, database, or publication that appears to reproduce or cite machine values.
- `observed`: measured play data, screenshots, counters, logs, or other observation.
- `inferred`: value derived from calculation, complement arithmetic, reverse solving, consistency constraints, or model fitting.
- `formula`: expression whose result depends on other rows or external constants.
- `assumption`: modelling assumption used only for scenario or sensitivity calculation.
- `unknown`: value not yet known or not yet accepted.
- `excluded`: value intentionally omitted from the model.

## Source priority policy

When multiple candidate sources exist, later PRs must record the source ordering used for selection.

Default priority order:

1. `primary`
2. `secondary`
3. `observed`
4. `inferred`
5. `formula`
6. `assumption`
7. `unknown`
8. `excluded`

The default order may be overridden only when the row records the reason.

Examples of override reasons:

- A primary source is incomplete for the specific route being modelled.
- A secondary source has clearer unit boundaries than an official promotional source.
- Observed data is used only to validate whether a cited value is plausible.
- An inferred value is only a complement of already accepted values.

## Citation metadata policy

Every later source-backed, formula-backed, inferred, candidate, or accepted row must preserve citation metadata or an explicit non-citation status.

Minimum citation fields:

- `sourceType`.
- `sourceTitle` or `sourceId`.
- `sourceLocation` when available.
- `retrievedAt` or `observedAt` when the source can change.
- `transcriptionStatus`.
- `confidenceStatus`.
- `notes` for conflict, exclusion, inference, or formula details.

Allowed transcription statuses:

- `not_required`.
- `machine_read`.
- `human_transcribed`.
- `human_transcribed_unverified`.
- `ocr_transcribed_unverified`.
- `derived_not_transcribed`.

Allowed confidence statuses:

- `accepted`.
- `provisional`.
- `conflicted`.
- `unverified`.
- `rejected`.
- `excluded`.

Rows without citation metadata must remain `required`, `stub`, `deferred`, `unknown`, or `excluded`.

## Probability unit policy

A probability row must declare its value representation and condition before it can be accepted.

Allowed probability representations:

- `ratio`: denominator-style notation such as 1/N.
- `percent`: percentage notation.
- `fraction`: numerator and denominator.
- `count_over_denominator`: observed count over total trials.
- `conditional_probability`: probability conditioned on route, state, event, or prior result.
- `complement_derived`: probability derived as the remainder after other probabilities.
- `formula_derived`: probability derived from a formula.

Required probability metadata:

- Representation.
- Numerator and denominator when applicable.
- Condition or route.
- Whether the probability is normalized.
- Whether the probability is mutually exclusive with sibling rows.
- Whether the probability is exhaustive within its parent route.
- Source metadata or inference metadata.

A probability row must not be accepted when its route condition is ambiguous.

## Reward unit policy

A reward row must declare its unit and accounting boundary before it can be accepted.

Allowed reward units:

- `payout_balls`.
- `net_balls`.
- `medals`.
- `currency`.
- `points`.
- `time`.
- `composite_expected_reward`.
- `formula_derived_reward`.

Required reward metadata:

- Unit.
- Whether the value is gross or net.
- Accounting boundary.
- Route or state condition.
- Whether the value is direct, inferred, or formula-derived.
- Source metadata or inference metadata.

A reward row must not be accepted when payout, net, and conversion boundaries are mixed.

## Inference policy

Inferred values must preserve their dependency chain.

Required inference metadata:

- `inferenceType`.
- Input row identifiers.
- Formula or derivation description.
- Whether all inputs are accepted.
- Whether any input is provisional.
- Whether the inferred row can be executable.

Allowed inference types:

- `complement`.
- `normalization`.
- `unit_conversion`.
- `route_split`.
- `reverse_solved`.
- `observed_estimate`.
- `consistency_check`.

An inferred row may become `accepted` only when all executable dependencies are accepted and the inference type is allowed for the row category.

## Formula policy

Formula-backed rows must preserve the expression, dependency list, and output unit.

Required formula metadata:

- Expression text or structured expression identifier.
- Dependency row identifiers.
- Output representation.
- Output unit.
- Evaluation boundary.
- Rounding policy.
- Whether the formula output is executable.

A formula-backed row must not be collapsed into a numeric constant without preserving the expression and dependencies.

## Conflict policy

Conflicting source values must not be resolved silently.

When conflicts exist, the row or candidate group must record:

- All known conflicting values.
- Source type for each value.
- Citation metadata for each value when available.
- Selected value, if any.
- Selection reason.
- Rejected or deferred value reason.
- Whether the conflict blocks execution.

A conflicted value must not become `accepted` unless the selection reason is recorded.

## Unknown, deferred, and excluded policy

Unknown, deferred, and excluded values must remain explicit.

- `unknown` means the value is not yet known or not yet inspected.
- `deferred` means the value is intentionally postponed.
- `excluded` means the value is intentionally outside the current model scope.

None of these statuses may be treated as zero.
None of these statuses may be treated as a default probability.
None of these statuses may be treated as a default reward.

## Promotion policy

A later row may be promoted to `accepted` only when the PR states the promotion boundary.

Required promotion statement:

- Row identifier.
- Previous status.
- New status.
- Source type.
- Citation metadata.
- Unit metadata.
- Inference or formula metadata when applicable.
- Conflict status.
- Execution impact.

Promoting a row to `accepted` is a value-inserting change, not a policy-only change.

## Review gates for later PRs

A later PR that adds or promotes real Beast King / Juo values must state which gate it satisfies.

Required gates:

- Machine scope fixed.
- Route inclusion and exclusion fixed.
- State list fixed or explicitly deferred.
- Transition graph fixed or explicitly deferred.
- Probability source metadata present for executable probability rows.
- Reward source metadata present for executable reward rows.
- Unit metadata present for executable rows.
- Inference dependencies present for inferred rows.
- Formula dependencies present for formula-backed rows.
- Conflicts recorded before selection.
- Unknown, deferred, and excluded rows remain non-executable.
