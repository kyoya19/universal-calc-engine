## Current guarantees

The current v4.x sequence guarantees the following:

- Requirements are expressed before production values are allowed.
- Missing fields must be explicitly represented by `required`, `stub`, `deferred`, or `excluded` statuses.
- State identifiers in the stub are unique.
- Exactly one start state is present in the stub.
- Exactly one terminal state is present in the stub.
- Transition references must point to existing state rows.
- Transition probability references must point to existing probability rows.
- Transition reward references must point to existing reward rows.
- Probability rows cannot silently gain executable probability values.
- Reward rows cannot silently gain executable reward values.
- Transition rows cannot silently gain executable probability or reward fields.
- Generic target semantics are still protected.
- Generic report identity is still protected.
