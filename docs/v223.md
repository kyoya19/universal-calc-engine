## Completed scope

The v22.x phase added a typed, non-executable metadata layer after materialization approval boundary rows.

The decision boundary layer now records probability and reward decision boundary rows linked to materialization approval boundary rows.

The completed decision boundary metadata covers:

- materialization decision boundary id
- materialization approval boundary id
- carried source, citation, unit, extracted value, draft, promotion, readiness, preflight, materialization, candidate, review, and approval metadata
- materialization decision eligibility
- materialization decision boundary status
- execution eligibility

Reward decision boundary rows additionally carry accounting consistency status.

## Current guarantee

A Juo source candidate can now pass through the materialization decision boundary without becoming executable.

At this checkpoint, source-backed value creation eligibility remains `no`, materialization eligibility remains `no`, materialization candidate eligibility remains `no`, candidate review eligibility remains `no`, approval eligibility remains `no`, materialization decision eligibility remains `no`, materialization decision boundary status remains `decision_blocked`, and execution eligibility remains `no`.
