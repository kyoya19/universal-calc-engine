## Purpose

This phase defines a guarded candidate review metadata boundary after guarded materialization candidate boundary rows and before any source-backed production value row can be created.

The phase is still metadata-only.

It does not insert real Beast King / Juo probability values or reward values.

## Candidate review boundary scope

A later guarded candidate review row may describe whether a blocked materialization candidate boundary row has passed review criteria for later production value materialization.

At this entry checkpoint, the candidate review boundary must remain non-executable and non-numeric.

The boundary is intended to separate:

- guarded source-backed production value materialization candidate boundary metadata
- guarded source-backed production value materialization candidate review metadata
- source-backed production value row creation
- executable production graph binding
- runtime target substitution
- expected value assertion

Only the first item is already present from v19.x.

The remaining items stay blocked.

## Required separation

A guarded materialization candidate review boundary row is not a production value row.

A candidate review boundary row may carry the metadata needed to decide whether later materialization review is still blocked or complete, but it must not contain:

- real Beast King / Juo probability values
- real Beast King / Juo reward values
- source-backed production value rows
- source-backed numeric values
- numerator values
- denominator values
- decimal values
- payout amounts
- production fixture values
- executable probability fields
- executable reward fields
- production graph bindings
- runtime target substitutions
- expected value assertions
- expected reward assertions
