## Purpose

This phase defines a guarded materialization candidate metadata boundary after source-backed production value materialization boundary rows and before any source-backed production value row can be created.

The phase is still metadata-only.

It does not insert real Beast King / Juo probability values or reward values.

## Candidate boundary scope

A later guarded materialization candidate row may describe whether a blocked materialization boundary row can be considered as a candidate for future source-backed production value creation.

At this entry checkpoint, the candidate boundary must remain non-executable and non-numeric.

The boundary is intended to separate:

- source-backed production value materialization boundary metadata
- guarded source-backed production value materialization candidate metadata
- source-backed production value row creation
- executable production graph binding
- runtime target substitution
- expected value assertion

Only the first item is already present from v18.x.

The remaining items stay blocked.

## Required separation

A guarded materialization candidate boundary row is not a production value row.

A candidate boundary row may carry the metadata needed to decide whether later materialization could be considered, but it must not contain:

- real Beast King / Juo probability values
- real Beast King / Juo reward values
- source-backed production value rows
- source-backed numeric values
- numerator values
- denominator values
- decimal values
- payout amounts
- production fixture values
- executable probability fields
- executable reward fields
- production graph bindings
- runtime target substitutions
- expected value assertions
- expected reward assertions
