## Purpose

This seed prepares the Juo PoC without introducing a machine-specific runtime implementation.

The immediate goal is to separate confirmed inputs, assumptions, unknowns, and out-of-scope items before any fixture contract or production behavior is added.

## Confirmed inputs

The following inputs are confirmed by the current project boundary only:

- Juo PoC is a machine-specific PoC candidate after the generic Sugoroku PoC boundary refresh.
- Juo PoC must reuse the generic state-transition model boundary.
- Juo PoC must not replace the existing generic expected reward baseline.
- Juo PoC must not wire `generatedTo` into solver or contribution targets.
- Juo PoC seed work should remain documentation-only or test-only until fixture expectations are explicit.
