## Shape invariants covered

The added test coverage verifies the current `ReportModel` shape contract used by UI-consumable reporting adapters.

The shared shape assertions cover:

- non-empty `ReportModel.kind`,
- non-empty `ReportModel.title`,
- at least one report section,
- unique section ids within a report,
- non-empty section ids,
- non-empty section titles,
- unique row ids within each section,
- non-empty row ids,
- non-empty row labels,
- non-empty row plain text fallback strings,
- row status values remaining inside the public `ReportRowStatus` union when present.

## Adapter structures covered

The current generated-target comparison adapter is covered for:

- stable section ordering: `summary`, `rows`,
- stable summary row ordering: `edgeCount`, `matchCount`, `missingGeneratedTargetCount`, `explicitGeneratedMismatchCount`,
- stable generated row ids using the `row-${index}` shape,
- plain text fallback structure generated from report title, section titles, and row plain text values.

The current generated-target solver gate summary adapter is covered for:

- stable accepted summary section ordering,
- stable accepted summary row ordering: `accepted`, `edgeCount`, `generatedTargetReadyEdgeCount`,
- stable rejected summary row ordering: `accepted`, `edgeCount`, `generatedTargetReadyEdgeCount`, `rejectionCode`, `rejectionType`, `rejectionMessage`,
- plain text fallback structure generated from report title, section title, and row plain text values.
