# project_scope_v0_3

## 結論

本リポジトリでは、サイコロPoCを最優先にする。

理由は、キヨタン／セイカタンへ進む前に、汎用確率状態遷移エンジンの最小処理線を確認する必要があるためである。

## 主対象

- 万能計算機そのもの
- 汎用確率状態遷移エンジン
- キヨタン＝順方向モデル
- セイカタン＝データからの逆方向推定モデル
- キヨタン／セイカタン往復評価

## 主対象外

- ツモロジ（仮）のマネタイズ企画全体
- 記事企画、編集部運用、課金、会員管理
- アプリ公開戦略
- 特許文面作成
- B2B営業設計

## 優先順序

1. サイコロPoC v0.3
2. 汎用モデル層の最小実装
3. OutputResult / ContributionResult の分離
4. 四号機獣王の公開解析値整合モデル
5. キヨタン順方向検証
6. セイカタン最小逆方向推定
7. キヨタン／セイカタン往復評価

## 判断

サイコロPoCは、すごろく計算機を作るためではなく、専用ロジックを混ぜずに状態遷移を展開・評価・解決できるかを確認するための最小検査である。
